# REQ-022 Planning Strategy Versioning and Audit

## User Story
As a planning maintainer,
I want planning strategies and priority policies to be versioned and auditable,
so that each plan decision is traceable to the strategy used.

## Acceptance Criteria
- Planning strategies include explicit version metadata.
- Priority policies include validity range and active status.
- Each plan references strategy and policy versions used at generation time.
- Historical strategy versions remain queryable for audits.

## Notes
This complements reasoning-side knowledge versioning.
