# REQ-027 Execution Command Graph for SPS/PLC

**User Story:**
As an execution engine (or industrial controller like SPS/PLC),
I want plans to be structured as a dependency graph rather than a flat sequence,
so that parallel actions and strictly sequential actions are explicitly defined and safely orchestrated.

**Acceptance Criteria:**
1. **Dependency Mapping:** Plan steps must include a `wait_for` or `depends_on` property linking them to previous step IDs.
2. **Machine Parsable:** Commands generated by the planner translate directly to executable technical payloads.
3. **Orchestration Friendly:** Flat/linear lists are avoided in favor of graphs to allow efficient execution where safely possible.
