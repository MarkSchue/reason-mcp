One thing I would like to think about:
Does it make sense to put our rules in a vector database and do semantic retrieval instead of deterministic retrieval?  This could the rule retrieval more flexible and robust to variations in the input.  We could still use deterministic retrieval as a first filter and then apply semantic retrieval on the database of rules to find the most relevant ones. I'm thinking about chunks of the natural language conditions, recommendations and keywords as the basis for the vector embeddings.  This way, we could capture the semantic meaning of the rules and match them to the context of the query more effectively.  However, this would also add complexity and latency to the retrieval process, so we would need to carefully evaluate the trade-offs and test it with real-world scenarios.
